Kernel Draw

Instructions
  • WARNING: all keywords must be lower case
  • create square id topLeftX topLeftY size (color: black, time: 5s)
  • create circle id centerX centerY radius (color: green, time: 5s)
  • create triangle id x1 y1 x2 y2 x3 y3 (color: red, time: 5s)
  • create intersection newID existingID_1 existingID_2 ... existingID_n (color: blue, time: 5s)
  • create union newID existingID_1 existingID_2 ... existingID_n (color: purple, time: 5s)
  • translate id translateX translateY (time: 2s)
  • scale id scaleFactor (time: 3s)
  • rotate id rotationAngleDegrees (time: 3s)
  • draw id (time: 1s)
  • wait coreID (no time required)
  • signal (no time required)
Notes
  • - The objective of the game is to create a drawing equal to the problem, while respecting the maximum allowed execution time.
  • - All 3 cores execute code in parallel. That being said you should be careful when writing instructions that act on the same variable, that might execute at similar time in different cores. If you want to assure synchronization/sequencialization we advise you to use wait/signal instructions (but be careful not create DEAD LOCKS)
  • - Each grid square has a size of 50 units
  • - The program execution time will be the one of the core with the longest execution
